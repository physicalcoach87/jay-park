RPE 기반 산출

01 sRPE Session RPE · 세션 운동강도 RPE ▼
단위AU (Arbitrary Unit)
데이터 출처session_records — rpe, duration
용도개인·팀 훈련 부하 정량화. ACWR 산출의 원재료
sRPE = RPE × 운동시간(분)

예) RPE 7 × 75분 = 525 AU

02 ACWR (개인) Acute:Chronic Workload Ratio RPE ▼
단위비율 (소수점 2자리)
데이터 출처session_records — rpe, duration, record_date (최근 28일)
용도부하 급증 감지 → CRI 입력값 · 부상 위험 예측
sRPE = RPE × 운동시간(분) Acute = 최근 7일 sRPE 합산 Chronic = (최근 28일 sRPE 합산 ÷ 유효 세션 수) × 7 ACWR = Acute ÷ Chronic
🔴 >1.5 급성 과부하 🟠 1.3~1.5 부하 주의 🟢 0.8~1.3 안전 구간 🟡 <0.8 과소부하

참고: Gabbett 2016

03 ACWR (팀 평균) Team ACWR 파생 ▼
용도팀 전체 부하 상태 파악 — 주간 현황 캘린더 상단 표시
각 선수 ACWR 개별 계산 (null 제외) 팀 ACWR = 유효 선수 ACWR 합 ÷ 유효 선수 수
※ 데이터 없는 선수(신입·부상 등)는 팀 평균에서 제외
04 주간 총 sRPE Weekly sRPE Sum RPE ▼
용도주간 훈련 부하 총량 확인 — 주간 현황 우측 표시
주간 총 sRPE = Σ (RPE × 운동시간) — 이번 주 전체 세션 합산

웰니스 기반 산출

05 웰니스 총점 Wellness Score (20점 만점) 웰니스 ▼
단위점 (0~20점)
입력 항목피로도 · 수면질 · 스트레스 · 근육통 (각 1~5점)
데이터 출처wellness_records — fatigue, sleep_quality, stress, muscle_soreness
용도일간 컨디션 모니터링 · CRI 입력값
총점 = 피로도 + 수면질 + 스트레스 + 근육통 (각 항목 1=최악 ↔ 5=최상)
🟢 17~20 최상 🟢 13~16 양호 🟡 9~12 주의 🔴 ≤8 위험/과훈련
06 웰니스 급락 감지 Wellness Drop Detection 웰니스 ▼
단위% (하락 비율)
용도단기 컨디션 급락 조기 감지 → CRI 입력값
7일 베이스라인 = 최근 7일 웰니스 총점 평균 dropPct = (베이스라인 − 오늘 총점) ÷ 베이스라인
🔴 ≥25% 급락 (+2점) 🟠 ≥15% 하락 (+1점)
07 체중 변화 (wDiff) Session Weight Change 웰니스 ▼
단위kg
데이터 출처wellness_records — morning_weight / session_records — post_weight
용도세션 중 수분 손실 감지
wDiff = 운동 후 체중(post_weight) − 아침 체중(morning_weight)
🔴 ≤−1.0kg 과도한 유수분 손실 🟠 −1.0~0kg 경미한 손실 🟢 ≥0kg 정상
08 7일 평균 체중 7-Day Average Morning Weight 웰니스 ▼
단위kg
용도오늘 체중과 7일 기준값 비교 — "오늘값/7일평균 kg" 형태로 표시
7일 평균 = 최근 7일 morning_weight 합산 ÷ 유효 일수 (오늘 제외)

GPS 기반 산출

09 HSR High Speed Running — 고속 러닝 거리 GPS ▼
단위m
데이터 출처gps_records — running_high(Band4), sprint(Band5)
용도팀·개인 고강도 이동거리 집계 · 주기화 목표 대비 실적
HSR = running_high (Band4) + sprint (Band5) ※ hsr_total 직접 필드 있을 경우 우선 사용
Band3 = 중속, Band4 = 고속(HSR), Band5 = 최고속(Sprint)
10 MPM Meters Per Minute — 분당 이동거리 GPS ▼
단위m/min
용도시간 정규화 강도 지표 · 출전시간 다른 선수 간 강도 비교 · 경기 롤링 분석 기초
MPM (td_per_min) = TD(총이동거리, m) ÷ 운동시간(분)
11 MSS 자동 갱신 Maximum Sprint Speed — 최고속도 자동 업데이트 GPS ▼
단위km/h
데이터 흐름gps_records.max_speed → player_profiles.mss 자동 갱신
갱신 시점GPS 파일 업로드 시
용도MV 노출 Zone5 기준값 · Match Max 최고속도 기준
업로드된 max_speed > 현재 MSS → MSS 자동 갱신 갱신 시 → 코치 알림 + 선수 개인 알림 자동 발송
12 MV 노출 Maximum Velocity Exposure — 최고속도 존 노출 분석 GPS ▼
단위일수, 비율(%)
데이터 출처gps_records — max_speed, session_date (최근 21일) / player_profiles — mss
용도햄스트링 부상 위험 감지 → CRI 입력값
Zone5 기준속도 = MSS × 0.9 exposureRatio = Zone5 도달 세션 수 ÷ 최근 21일 총 세션 수 daysSinceExposed = 마지막 Zone5 노출일로부터 경과 일수 pctMSS = 최근 7일 최고속도 ÷ MSS × 100
🔴 14일↑ 미노출 햄스트링 고위험 (+3점) 🟠 7~13일 미노출 주의 (+2점) 🟡 pctMSS <80% 속도 노출 부족 (+1점)

참고: Malone 2017, Duhig 2016

13 표준편차 · 선수 편차 Standard Deviation — 팀 내 개인 편차 GPS ▼
용도선수별 GPS 지표가 팀 평균에서 얼마나 벗어나는지 표시 (개인 상세 테이블)
표준편차(SD) = √[ Σ(값 - 평균)² ÷ 선수 수 ] 개인 편차(%) = (개인값 - 팀 평균) ÷ 팀 평균 × 100 ※ |편차| ≥ 10% 이상일 때 +/- 뱃지 표시

팀 평균 산출 (3종)

14 훈련 팀 평균 (trainAvg) 최빈 훈련시간 기준 정규화 GPS ▼
적용 조건session_type2 = 'M' 인 선수만 (주전·엔트리)
용도데일리 브리핑 · 주간 현황 · 목표 대비 실적 집계
최빈 훈련시간(mode) = 가장 많은 M 선수가 공유하는 훈련시간 trainAvg = (M선수 지표 총합 ÷ M선수 훈련시간 총합) × 최빈 훈련시간
사용 이유: 부상·로테이션으로 일부 선수가 짧게 훈련했을 때 단순 평균 왜곡 방지.
M 있는 날 → M 기준 / M 없는 날 → S 기준 평균 사용
15 경기 팀 평균 (matchAvg) 최대 출전시간 기준 정규화 경기 ▼
적용 데이터match_records, time_type = 'FT'
용도최근 3경기 기준선 · 경기 보고서 팀 비교
matchAvg = (선수 지표 총합 ÷ 출전시간 총합) × 최대 출전시간(max duration)
사용 이유: 교체 선수 출전시간이 짧아 지표가 낮아지는 문제 보정.
최장 출전자(풀타임 기준) 시간으로 환산하여 팀 레벨 비교 가능하게 함
16 팀 평균 /100분 (p100) 100분 기준 정규화 강도 지표 파생 ▼
용도경기 보고서 — 출전시간이 다른 선수들을 100분 기준으로 균등화 비교
p100 = 지표 총합 ÷ 출전시간 합계 × 100
17 최근 3경기 팀 기준선 (ref3) Match Reference — 훈련 강도 비교 기준 경기 ▼
산출 지표TD · HSR(Band4) · Sprint(Band5) · 가속 · RHIE
산출 방법최근 3경기 FT 전체 선수 데이터 → matchAvg 공식 적용
용도훈련 데이터 옆 % 뱃지 기준 — "오늘 훈련이 경기 대비 몇 %인가"
ref3 = matchAvg(최근 3경기 FT 전체 선수 데이터, 지표) 뱃지% = 훈련 실제값 ÷ ref3 × 100
🟢 ≥80% 충분 🟡 60~79% 주의 🔴 <60% 부족

경기 데이터 산출

19 Match Max 경기 최대 퍼포먼스 기준값 (커리어 최고) 경기 ▼
산출 지표TD · MPM · HIR+Sprint · HSR(Band4) · Sprint(Band5) · 가속 · RHIE · 최고속도 · Player Load · FMP 4종
저장 위치player_profiles — match_max_* 필드 (14개)
갱신 시점경기 데이터 업로드 시 / 전체 재계산 버튼
① FT 90분↑ 경기 ≥3개
상위 3개 raw 값 평균
② FT 90분↑ 경기 1~2개
raw 값 + 60~89분 경기를 출전시간 많은 순으로 보충 → 정규화(÷출전시간×100) 후 3개 평균
③ 90분↑ 없고 60분↑ 있음
출전시간 많은 순 3경기 정규화 평균
④ 60분 미만만 있음
null (미산출)
정규화값 = 지표값 ÷ 출전시간(분) × 100 최고속도(MSS) = max(경기 max_speed, 훈련 GPS max_speed) ← 단일 최대값
20 절대% · 강도% 개인 Match Max 대비 경기 퍼포먼스 경기 ▼
용도경기 보고서 — 이 경기가 커리어 최고 대비 어느 수준인가
절대% (총량 기준) = 이 경기 TD ÷ Match Max TD × 100 강도% (시간 기준) = 이 경기 MPM ÷ Match Max MPM × 100
★ ≥90% 커리어 최고 수준 — 70~89% 정상 범위 ↓ <70% 저하
21 경기 팀 합계 · 경기당 팀 평균 teamSum / teamAvg 경기 ▼
용도경기 보고서 팀 섹션 — FT·1Q·2Q·Top-up 각각 집계
팀 합계(f) = Σ 선수 지표값 (양수인 선수만 합산) 단순 평균(avg) = Σ 지표값 ÷ 선수 수 p100 강도 = Σ 지표값 ÷ Σ 출전시간 × 100 팀 평균(norm) = Σ 지표값 ÷ Σ 출전시간 × max 출전시간
경기 보고서는 합계(절대량)와 팀 평균(정규화) 두 버전 모두 표시
22 전후반 비교 1Q vs 2Q / Halftime Auto-detection 경기 ▼
데이터match_records — time_type: '1Q' / '2Q'
비교 지표TD · HIR+Sprint · 평균 MPM · 가속 · 감속 등
용도후반 강도 저하(PIP 손실) 감지 → 회복 훈련 처방 근거
mpm 감소율 = (2Q mpm − 1Q mpm) ÷ 1Q mpm × 100 하프타임 자동 = FT 선수 평균 mpm에서 전후반 20~80% 구간 내 최대 갭(≥3분 단절) 자동 탐지 2Q 종료 자동 = 하프타임 이후 팀 평균 mpm ≥ 30인 마지막 분
PIP 손실 판정: 후반 동일 구간 mpm이 전반 평균 대비 −10% 이하 → 회복 훈련 처방 우선
23 mpm 롤링 분석 (WCS) Peak Intensity / Worst Case Scenario 경기 ▼
데이터분당 mpm — mpm 파일 업로드로 생성, FT 출전 선수만 포함
용도훈련 처방 근거 — 경기 최고 강도 구간 수치화 → 훈련에서 재현
Peak 1분 = 연속 1분 구간 mpm 평균 중 최대값 Peak 5분 = 연속 5분 구간 mpm 평균 중 최대값 (슬라이딩) Peak 10분 = 연속 10분 구간 mpm 평균 중 최대값 (슬라이딩) 체류시간 = mpm > 임계값(150/130/110 m/min)인 분수 합계
훈련 처방 목표: Peak 5분 mpm의 90% 이상 복제
체류시간 >150 m/min → 인터벌 처방 강도 기준

지표 4종FMP Run Medium · FMP Run High · FMP Dynamic Medium · FMP Dynamic High (단위: 초)
